Isto aqui era um vb. do EV no tempo em que o OZ funcionava lá (q.v., se precisar trabalhar naquela versão, o "texto obsoleto sobre EV" abaixo). 

Agora, por enquanto, a explicação sobre o que cada tag abrange e a qual vb. indexador ela se refere está sempre no dito vb. indexador (por exemplo, para saber o que significa a tag #proj_zetteltex  tem de ver a explicação no vb. [[Projeto-Zetteltex-AKA-OZ-index\|Projeto-Zetteltex-AKA-OZ-index]]). Ou seja, cada zettel-index é na verdade uma tag em formato de zettel. O sistema se baseia nas tags, fundamentalmente, para classificar / catalogar / organizar / indexar o zettelkasten. Os zettels-indexadores são apenas materializações das tags em formato de texto e servem para recepcionar as listas de remissões que o Obsidiam gerar (e para explicar o que a tag abrange).

Para saber qual o verbete-indexador de uma tag, procure por essa tag no zettel [[RQR rol index grandes temas ou questoes recorrentes fichas principais\|RQR rol index grandes temas ou questoes recorrentes fichas principais]]. 

Está bom assim, vamos por neste zettel só o que for genérico ou se referir a tags "classificatórias" que não correspondem direta e exclusivamente a um vb. indexador, como `star`, `vip`, `nsfw` etc.

## convenções sobre tags no OZ

**A convenção mais importante**: toda vez que criar uma tag (não meramente classificatória), criar um zettel-index para ela e anotar esse par no [[RQR rol index grandes temas ou questoes recorrentes fichas principais\|índice de RQRs]]. Se for classificatória, em vez disso faça uma anotação neste zettel, logo abaixo.

**Tags de classificação**. Não correspondem a um zettel-indexador, normalmente. São adicionadas a zettels para classificá-los em classes como:

1. **indexes**: tomar cuidado e não abusar dessa. Por só nos zettels que correspondem diretamente a um index de tag ou que corporificam uma tag. Todo (?) index está também com a tag RQR, que parece ter ficado supérflua...
2. **consertar**: o que o nome diz. 
3. **apagar**: depois da tag deve vir uma data estimada, para ser lida nos resultados de pesquisa sem precisar abrir o zettel
4. **arquivo-morto**: poderia apagar, mas vai ficando porque pode ter interesse documental.
5. **task**: para ser lida pelos plugins gestores de tasklists. Em princípio qualquer e todo zettel pode conter uma task, ainda que temporária. Mas haverá um zettel para agrupar tarefas do trabalho [[251205_S_Tasks_Work\|251205_S_Tasks_Work]] e outro para tarefas genéricas [[251205_S_Tasks_Misc\|251205_S_Tasks_Misc]]. Zettels genéricos podem conter tasks relativas especificamente ao assunto deles. *Importantes convenções/workflows* sobre isso: **(a)** coloque a tag no zettel, não na tarefa, a menos que só haja uma tarefa no zettel; **(b)** apague a tag quando a tarefa estiver cumprida. 
6. **inbox**: para marcar/bookmarkar os zettels que estão em uso no momento, uma espécie de topo de mesa para zettels.
7. **reading_list**: o que o nome indica.
8. **urgente**: o que o nome diz, bom colocar um apagar junto.
9. **bora-fazer**: ideias para futuros projetos.
10. **admin**: tag supérflua, podemos deixar só 'internas' ( #bora-fazer ).
11. **internas**: zettels de gestão do Zettelkasten; está mal usada, praticamente tudo que é "ajuda" está com essa tag, e não devia.
12. **nsfw**: para facilitar a eliminação de zettels críticos. Não aplicar essa tag em ziniaz.
13. **cool, wtf, ooommm, oops, lol**, **espantos**, **antologia**: para comentar frases que estão em zettels indexados alhures.
14. **ziniaz**: porque uma zinia pode estar misturada num zettel sobre um contrato ou compra.
15. **Work**: embora exista um [[27G68a workstuff _index_\|27G68a workstuff _index_]], muitas coisas entram na classe Work sem merecer ser indexadas. O mesmo vale para **contas**, **rolos**, **bizz**, **docs**. 
16. **BC3**: coisas do trabalho que deveria ir para a BC3 se eu tivesse saco. Não tem index?
17. **RC**: porque notas sobre coisas lidas no dia a dia podem estar em zettels de vários tipos, por exemplo Journals.
18. **Journal**: porque entradas do tipo Journal podem aparecer mixadas em zettels de resumo de coisas lidas, etc.
19. **conjuri24** ou `#proj_PABAN`: é um exemplo de tag-de-projeto; muito embora cada projeto deva ter um zettel-indexador... Então talvez isso não seja exatamente tag classificatória.

**Diferença entre as tags STAR  e VIP:** Usa STAR para verbetes muito bons que merecem ser relidos (uma espécie de pérolas); usa VIP para verbetes que são importantes mas não são de reler, são coisas que posso precisar um dia e frequentemente entram na classe _documento_ ou na classe _internas_. Equivale a um "nunca-apagar".

**Verbetes centralizadores de tags cruciais** a serem revistas periodicamente: listam os zettels que contêm certas tags relevantes para a gestão do OZ:

- [[251214_S_zettels_para _consertar_index\|251214_S_zettels_para _consertar_index]]
- [[251214_S_zettels_para _apagar_index\|251214_S_zettels_para _apagar_index]]
- [[251214_S_zettels_com_TASKS_pendentes_index\|251214_S_zettels_com_TASKS_pendentes_index]] — este é importante porque lista todas as tarefas pendentes, é gestão de agenda e não do OZ propriamente.
